在 Windows 下用U盘安装 openSUSE

2014-04-15

不使用光盘安装系统已经是大势所趋。Ubuntu系的iso文件刻录到U盘安装相当容易。但是想装个openSUSE却费了好多事。记录以供后人参考。

系统硬件：

  • ASUS 的主板，BIOS是那种图形化的，进去能用鼠标操作的。但是貌似也不是UEFI。没搞清楚
  • 硬盘已经安装了Win7、Win8和Ubuntu(尝试安装的过程中格式化了)
  • 一枚2G以上的U盘。

安装方法：

  1. 从openSUSE的网站上下载安装镜像，我使用的是KDE Live镜像。

  2. 下载安装Win32DiskImager这个软件。注意，Ultraiso、Unetbootin亲测不好使。EasyBCD的光盘引导方式也不行。Linux下添加Grub启动项也不行(可能是我姿势不对)。各位不要作无谓的挣扎了。

    Linux下用dd命令应该是可以的，我觉得。未亲测

  3. 刻录。重启，从U盘启动。一般是开机时按F8出现启动选择菜单。或者进BIOS自行设置

    注意，ASUS主板的启动菜单里U盘会显示出两个，一个是xxx(U盘名)，另一个是UEFI xxx(U盘名)。要选不带UEFI的。

  4. 正常安装。

以下是吐槽部分：

之前使用的是Linux Mint的KDE系统，挺漂亮的。但是源不快啊！！ 搞不明白这帮发行版为什么要自己搞软件管理器。Apper不是很好么！嗯，当然重装的原因大概是我添加了Kubuntu的backports源，造成了软件依赖冲突，安装软件总是报错，关于python-sid的，某某脚本错误什么的。

本来的选择是换回Kubuntu。但是装完这货发现磁盘奇慢无比。不知道哪里的问题。以前遇到过，好像一度启动失败，检查了下磁盘坏块才进去。看来这货没有反省自己。安装完中文字体没有，中文是诡异的点阵字体，装完文泉驿字体设置后还有一些GUI是这样。而且，Kubuntu改用了非常非常不好用的Moun软件管理器，找包完全不能，而且必须事先知道包在哪个分类才行。好吧。Kubuntu再见。

然后想试试Fedora。这货U盘启动支持还挺好的(用它自家的刻录工具行，用Ultraiso也启动不了，好像)，能进桌面。但是安装时卡在了磁盘分区这里。我明明给它指定了/分区和/home分区，这货却呆萌的说我的磁盘没有空间。怎么哄都无济于事。好吧。看来是不喜欢我的磁盘。那我只好放弃你了。(估计这是Fedora使用了新的安装程序造成的。没事瞎TMD折腾啥。一点也没看出新安装器有啥好来。)

然后想装文明用语试试Arch。但是这货的安装iso根本不支持U盘刻录安装。启动时能进命令行，但是安装指南中所说的各种命令根本不能使。好像尝试用了官方推荐的dd方法刻录也不行。好吧，我的计算机这么奇葩吗？您这样倔强的不愿意将就？

好吧。尝试业内传说的最华丽的发行版openSUSE吧。其实在Fedora之前我就试了它了。但是用前面说的各种工具刻录或启动，均失败。启动时显示

啥啥啥的。找了半天网上的帮助，也没说出个啥来。

心中骂了无数遍。后来抱着试试看的心理用了Win32DiskImager，靠，居然可以了。心里不知道是什么滋味。下面就很顺利了。openSUSE的安装程序很好很漂亮。心动了的说。尤其强大的是它直接选择了/分区默认格式化、/home分区默认不格式化，太强大了。这一点绝对要点赞！

适应openSUSE中。
